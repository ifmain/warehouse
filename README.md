# Склад Техники

Небольшое веб-приложение для учета товаров на складе магазина техники.

## Стек

* **Бэкенд:** ASP.NET Core API (.NET 10), EF Core
* **Фронтенд:** Blazor WebAssembly (.NET 10), Refit
* **База данных:** PostgreSQL 15
* **Реалтайм:** SignalR
* **Инфраструктура:** Docker, Docker Compose, Nginx

## Что внутри

Проект разбит на три части:

* `warehouse.API` — REST API, работа с БД, сохранение картинок локально и SignalR хаб.
* `warehouse.Blazor` — SPA-клиент на WebAssembly. Общается с бэком через Refit.
* `warehouse.Shared` — общая либа с DTO-моделями и контрактами.

## Что умеет

* **Учет товаров**. Стандартный CRUD. Можно грузить фотки. Если при добавлении товара нужной страны нет, ее можно создать прямо в форме.
* **Справочники**. Отдельное управление производителями, типами техники и странами.
* **Фильтры и поиск**. Фильтры по типу и бренду связаны: если выбираешь конкретный бренд, в типах останутся только те категории, которые есть у этого бренда на складе (и наоборот). Есть поиск по тексту (модель, тип, бренд) и фильтр по цене.
* **Синхронизация**. Через SignalR. Если кто-то добавил, изменил или удалил товар или справочник, данные обновятся у всех пользователей сразу, без перезагрузки страницы.
* **Интерфейс**. Есть пагинация. Каталог можно смотреть таблицей или карточками (выбор сохраняется в `localStorage`). Формы и фотки открываются в модалках.

## Запуск

Для локального запуска нужны только Docker и Docker Compose.

```bash
git clone <url_репозитория>
cd warehouse
docker-compose up -d --build

```

После старта контейнеров:

* Фронт: `http://localhost:80`
* API: `http://localhost:5213`
* БД: `localhost:5432`

База и таблицы создаются автоматически при старте API (через `EnsureCreated()`). Картинки сохраняются в папку `uploads_data` на хосте, поэтому при пересоздании контейнеров они не удалятся.
